# Strapi Setup & Deployment Guide

This project uses Strapi as a headless CMS with PostgreSQL configured for production. Below are the steps to set up, configure, and run the application.

---

## 🚀 Project Setup

Create a new Strapi project using the official CLI:

```bash
npx create-strapi-app@latest my-project
cd my-project
```

---

## 🛠️ Install Dependencies

Install the PostgreSQL driver required for production:

```bash
npm install pg
```

---

## ⚙️ Environment Configuration

Create a `.env` file in the root directory and configure it as follows:

```env
# Server
HOST=0.0.0.0
PORT=1337

# Database (PostgreSQL)
DATABASE_CLIENT=postgres
DATABASE_HOST=127.0.0.1
DATABASE_PORT=5432
DATABASE_NAME=strapi
DATABASE_USERNAME=strapi_user
DATABASE_PASSWORD=your_password
DATABASE_SCHEMA=public
DATABASE_CONNECTION_TIMEOUT=60000

# Secrets (auto-generated by Strapi)
APP_KEYS=your_app_keys
API_TOKEN_SALT=your_api_token_salt
ADMIN_JWT_SECRET=your_admin_jwt_secret
TRANSFER_TOKEN_SALT=your_transfer_token_salt
ENCRYPTION_KEY=your_encryption_key
JWT_SECRET=your_jwt_secret
```

> ⚠️ Important: Keep all secret keys secure and do not commit them to version control.

---

## ▶️ Running the Application

### Development Mode

```bash
npm run develop
```

Access admin panel: http://localhost:1337/admin

---

### Production Mode

```bash
npm run build
npm run start
```

---

## 🗄️ Database Configuration

- Default: SQLite (development)
- Production: PostgreSQL (configured via `.env`)
- Ensure PostgreSQL is running before starting the app
- Update credentials based on your database setup

---

## 🔐 Permissions Setup

1. Go to **Settings → Roles & Permissions**
2. Select role (Public or Authenticated)
3. Enable:
   - POST
   - PUT
   - DELETE

---

## 📦 Notes

- Restart the server after updating `.env`
- Ensure environment variables are correctly set before deployment
- Do not expose sensitive credentials publicly
